SQL은 무엇의 약자
Structured Query Language

SQL의 3가지 비교 설명한 것을 예로 드시오
Data Definition Language(DDL): 데이터 정의 언어 (CREATE, ALTER, DROP)
Data Manipulate Language(DML): 데이터 조작 언어 (SELECT, INSERT, UPDATE, DELETE)
Data Control Language(DCL): 데이터 제어 언어 (GRANT, REVOKE)

SQL에서 join이란
두 테이블 사이에서 특정 column을 기준으로 두 테이블을 결합하는 것

SQL join의 종류
inner join: 조건에 맞는 record만 결합
left join/right join: 왼쪽/오른쪽의 테이블은 조건에 맞지 않아도 결합에 참여

PK와 FK는 무엇의 약자 어떨 때 사용하는가
Primary Key: 기본키 레코드를 식별할 때 사용하는 고유값
Foreign Key: 외래키 다른 테이블의 기본 키를 가리키는

무결성이란
개체무결성: 모든 테이블은 기본 키를 갖는다
참조무결성: 모든 외래 키는 릴레이션의 기본키이거나 NULL이다 

DB에서 이상현상이 왜 발생하는가? 이상현상이 발생하는 경우 3가지
DB설계가 잘못되어 데이터가 중복될 경우 발생하게 된다
삭제이상: 레코드를 삭제할 때 다른 정보를 잃어버리게 되는 현상
삽입이상: 레코드에 불필요한 정보를 넣지 않으면 필요한 정보를 넣을 수 없게 되는 현상
갱신이상: 정보를 갱신해야 할 때 중복된 레코드의 개수만큼 갱신이 필요한 현상

이상현상의 해결책은? 몇 가지 단계가 있는가? 3가지 이상
제1정규화: 중복되는 항목이 없다.
제2정규화: 모든 속성은 후보 키에 대해 함수 종속이다.
제3정규화: 모든 속성이 후보 키가 아닌 다른 속성에 이행적 종속되지 않는다.

자료구조에서 STACK과 QUEUE를 비교 설명하시오
STACK: 먼저 들어온 자료가 나중에 나가는 구조
QUEUE: 먼저 들어온 자료가 먼저 나가는 구조

배열, 리스트, 맵 비교 설명
배열: index와 값으로 이루어짐
리스트: 값과 다음 데이터 위치로 이루어짐
맵: 키와 값으로 이루어짐

ORM은 무엇인가?
Object-Relation Mapping: DB 쿼리의 결과를 객체로 변환하는 것
ORM의 예로는 MyBatis, Hibernate 등이 있다

스레드란? 멀티스레드란?
스레드: 작업을 수행하기 위한 단위
멀티스레드: 하나의 프로세스가 여러 개의 스레드를 사용하는 것

DAO, DTO는 무엇인가?
DAO: Data Access Object: DB에 쿼리를 전달하기 위해 사용하는 객체
DTO: Data Transfer Object: 

결합도는 무엇인가?
모듈이 다른 모듈에 의존하는 정도
결합도가 높으면 모듈을 교체할 때 수정해야 하는 부분이 많아지므로 낮은 편이 바람직하다.

응집도는 무엇인가?
모듈 안의 구성요소가 밀접한 정도
응집도가 낮으면 한 가지 모듈이 여러 일을 하게 되므로 높은 편이 바람직하다.

final class, final method, final 변수를 설명하시오
final class: 더 이상 상속이 되지 않는 클래스
final method: 더 이상 오버라이딩할 수 없는 메서드
final 변수: 재정의할 수 없는 변수

static 변수와 instance 사이의 차이를 설명하시오
static 변수는 컴파일할 때 메모리의 스태틱 영역에 할당되는 변수로 스코프 내부에서 변수값 재할당이 일어났을 때 스코프를 벗어나도 그 값이 유지된다.
instance는 선언 타이밍에 힙 영역에 할당되는 변수로 스코프를 벗어나면 값이 소멸한다.

원시자료형과 참조자료형의 차이
원시자료형은 언어에서 지원하는 기본 자료형으로 정수, 실수, 문자, 진리값이 있다.
참조자료형은 다른 자료형이나 메서드의 복합으로 나타내는 자료형으로 String. Date 등이 있다.

Wrapper class는 무엇인가?
Wrapper class는 원시자료형을 객체처럼 다룰 필요가 있을 때 사용하는 클래스이다
int의 wrapper인 Integer, double의 wrapper인 Double, char의 wrapper인 Character 등이 있다.

TCP, UDP의 차이는 무엇인가?
TCP는 두 단말이 연결되어 서로 패킷을 전송하는 방식으로 신뢰성이 보장된다.
UDP는 한 단말이 불특정 다수에게 패킷을 전송하는 방식으로 성능이 보장된다.

TCP/IP 4단계 구분, 설명, 예시
네트워크액세스계층: 물리계층+데이터링크계층. 실제 데이터가 MAC주소를 따라 통신노드를 오간다.
인터넷계층: 네트워크계층. 통신 노드 간에 IP 패킷을 전송하고 라우팅한다.
전송계층: 통신 노드 간의 연결을 제어한다.
응용계층: TCP 연결 기반으로 응용 프로그램을 구현한다.

싱글톤이란? 프로토타입과 싱글톤의 차이
싱글톤은 생성자가 여러번 호출되어도 객체가 하나만 생성되도록 관리하는 프로그래밍 패턴이다
프로토타입은 생성자가 호출될 때마다 객체가 생성된다
